## KeyPilot Monorepo Scaffolding Brief

Read these files first before generating any code:

1. `START_HERE_FOR_AI.md`
2. `AI_CONTEXT.md`
3. `SYSTEM_DESIGN.md`
4. `CODING_GUIDELINES.md`
5. `docs/architecture/overview.md`
6. `docs/architecture/modules.md`

These files define the architecture, domain model, and coding rules for KeyPilot.

---

## Objective

Scaffold a runnable monorepo for **KeyPilot** with:

- `frontend/` → Next.js 15 + TypeScript + Tailwind + App Router
- `backend/` → .NET 10 Web API modular monolith
- local Postgres via Docker Compose
- minimal working vertical slice for `Property`

Do **not** add enterprise complexity beyond what is needed to run locally and support the MVP architecture.

---

## Architectural Constraints

### Required
- Modular monolith
- Mobile-first frontend
- REST API
- Postgres as primary database
- EF Core for persistence
- Swagger enabled in development
- Clean folder structure aligned to future domain growth

### Explicitly do not add
- Microservices
- Kafka or message bus
- Generic workflow engine
- Event sourcing
- Auth implementation
- Document upload implementation beyond placeholders
- Over-engineered shared libraries

## Backend Requirements

### Solution layout
Create a .NET 10 solution named `KeyPilot` with these projects:

- `KeyPilot.Api`
- `KeyPilot.Application`
- `KeyPilot.Domain`
- `KeyPilot.Infrastructure`
- `KeyPilot.Api.Tests`

### Project references
Use this dependency direction:

- `KeyPilot.Api` → `KeyPilot.Application`, `KeyPilot.Infrastructure`
- `KeyPilot.Application` → `KeyPilot.Domain`
- `KeyPilot.Infrastructure` → `KeyPilot.Application`, `KeyPilot.Domain`
- `KeyPilot.Api.Tests` → `KeyPilot.Api`

### Internal backend structure

#### `KeyPilot.Domain`
```text
Domain/
  Common/
    Entity.cs
    AuditableEntity.cs
  Properties/
    Property.cs
    PropertyStatus.cs
```


#### `KeyPilot.Infrastructure`

#### `KeyPilot.Api`

### Backend packages

#### Api
- `Swashbuckle.AspNetCore`
- `Microsoft.AspNetCore.OpenApi`

#### Application
- `MediatR`
- `FluentValidation`

#### Infrastructure
- `Microsoft.EntityFrameworkCore`
- `Microsoft.EntityFrameworkCore.Design`
- `Microsoft.EntityFrameworkCore.Relational`
- `Npgsql.EntityFrameworkCore.PostgreSQL`

#### Tests
- `Microsoft.AspNetCore.Mvc.Testing`
- `FluentAssertions`

### Backend feature scope
Implement only:

- `GET /health`
- `POST /api/v1/properties`
- `GET /api/v1/properties/{id}`

### Property entity fields
Implement a simple v1 `Property` model with:

- `id`
- `workspace_id` nullable for now
- `address`
- `status`
- `offer_accepted_date`
- `settlement_date`
- `purchase_price` nullable
- `created_at_utc`

### Property status values
Use enum or string-backed enum with:

- `draft`
- `conditional`
- `unconditional`
- `settled`

### Backend runtime behaviour
Wire the backend to:

- Npgsql / Postgres
- Swagger in development
- CORS allowing `http://localhost:3000`
- health checks
- dependency injection for Application and Infrastructure

### Database
Use EF Core with Postgres.
Make migration support ready, even if the first migration is not fully generated by the agent.

## Scaffolding Commands Reference

### Frontend
```bash
mkdir -p frontend
cd frontend
npx create-next-app@latest . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*" --use-npm
npm install zod react-hook-form @hookform/resolvers lucide-react clsx tailwind-merge
npm install @tanstack/react-query
npm install axios date-fns
npx shadcn@latest init
npx shadcn@latest add button card input form badge separator sheet
```

### Backend
```bash
mkdir -p backend/src
mkdir -p backend/tests
cd backend
dotnet new sln -n KeyPilot

dotnet new webapi -n KeyPilot.Api -o src/KeyPilot.Api
dotnet new classlib -n KeyPilot.Application -o src/KeyPilot.Application
dotnet new classlib -n KeyPilot.Domain -o src/KeyPilot.Domain
dotnet new classlib -n KeyPilot.Infrastructure -o src/KeyPilot.Infrastructure
dotnet new xunit -n KeyPilot.Api.Tests -o tests/KeyPilot.Api.Tests

dotnet sln KeyPilot.sln add src/KeyPilot.Api/KeyPilot.Api.csproj
dotnet sln KeyPilot.sln add src/KeyPilot.Application/KeyPilot.Application.csproj
dotnet sln KeyPilot.sln add src/KeyPilot.Domain/KeyPilot.Domain.csproj
dotnet sln KeyPilot.sln add src/KeyPilot.Infrastructure/KeyPilot.Infrastructure.csproj
dotnet sln KeyPilot.sln add tests/KeyPilot.Api.Tests/KeyPilot.Api.Tests.csproj

dotnet add src/KeyPilot.Api/KeyPilot.Api.csproj reference src/KeyPilot.Application/KeyPilot.Application.csproj
```

## Acceptance Criteria

The scaffolding is complete when all of the following are true:

1. `docker compose up -d` starts Postgres successfully
2. backend builds successfully
3. backend runs on `http://localhost:8080`
4. Swagger loads in development
5. `GET /health` returns success
6. `POST /api/v1/properties` creates a property
7. `GET /api/v1/properties/{id}` returns that property
8. frontend runs on `http://localhost:3000`
9. the create-property form submits successfully to the backend
10. the property details page loads the created property

## Implementation Notes for the Agent

- Prefer simple, readable code over abstractions.
- Keep the backend runnable as a single deployable API.
- Keep the frontend mobile-first and clean.
- Add placeholder comments where future modules will live.
- Do not invent extra product scope.
- Focus on one working vertical slice through the system.
